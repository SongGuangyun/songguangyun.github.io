使用vagrant打造统一的开发环境

简介

vagrant是一个构建和管理虚拟机环境的工具，我们可以在vagrant上搭建好开发环境，然后打包分发给你的团队成员，实现环境统一，从而不会出现”代码在我的机子上运行没问题啊”的问题

安装

ps:以下安装均在win10环境下
　　vagrant的安装：点进这里，看到windows的就点击下载
　　VirtualBox的安装：点进这里，看到Windwos hosts点击下载就好
注意：vagrant的版本和virtual box的版本不对应也会出现不知名的错误，我这里的的vagrant版本为1.8.6,virtual box版本为5.1.22
vagrant选择版本：点击这里

创建好开发环境目录之后，我们该做什么？

对了，我们需要box，有了box我们才能启动vagrant虚拟机，那box怎么来？就是上面box add,所以我们就需要.box镜像文件！
这里有两种情况：
　　1.一是我们本机有box文件，这种情况我们直接进入开发环境目录中：box add ubuntu_song ubuntu16.04.box。注意：如果你是先box init,再box add，那么你要注意把config.vm.box 选项对应的值改成你的box名称，也就是上面的ubuntu_song，然后vagrant up启动，启动后查看状态，vagrant status，如果 running 证明启动成功！
　　2.第二种情况是没有box文件（没有box镜像就先不要vagrant init），我们可以：vagrant init ubuntu/trusty64，然后vagrant up –provider virtualbox，这个是让云服务器提供virtualbox版本的ubuntu/trusty64（同理，也有不是virtualbox版本的，下载时要注意不要下错版本），接着，你就会看到它在自动下载.box文件，如果你觉得下载速度慢，可以将链接复制到迅雷，这样就可以把镜像下载到本地，例如点击这里可以下载ubuntu/trusty64的镜像
　　3.其实原理就是进到云服务器，搜索你要的镜像，例如centos7，点进去可以看到两条命令，vagrant init centos/7; vagrant up –provider virtualbox，你要做的就是进入一个目录，依次执行这两条命令，下载完后，同情况一，启动：vagrant up

启动虚拟机后，我们该做什么

　　登录：vagrant ssh
　　退出：exit
　　关于一些常见的vagrant状态（当你vagrant status之后可以看到）
　　running (virtualbox) up后
　　poweroff (virtualbox) halt后
　　saved (virtualbox) suspend后
　　not created (virtualbox) destroy后
